### 4️⃣ **История актива** — 1.5 часа
**Зависит от:** ничего (но логично после экспорта)  
**Влияет на:** Asset Management

**Что делать:**
- Backend:
  1. Новая коллекция `asset_history` (asset_id, event_type, old_value, new_value, user_id, created_at)
  2. При `PUT /assets/{id}` → логировать изменения в history (если mol_user_id, location, status изменились)
  3. Новый эндпоинт `GET /assets/{id}/history` → вернуть список событий
  4. Миграция: для старых активов создать запись "Created" в history

- Frontend:
  1. На странице редактирования актива добавить вкладку "История"
  2. Список событий (дата, кто, что было → что стало)

**Структура записи history:**
```python
{
  "asset_id": ObjectId,
  "event_type": "created" | "mol_changed" | "location_changed" | "status_changed" | "repaired",
  "old_value": {
    "mol_user_id": "...",
    "location": "..."
  },
  "new_value": {
    "mol_user_id": "...",
    "location": "..."
  },
  "user_id": ObjectId,
  "created_at": ISODate
}
```

**Тестировать:**
- Изменить МОЛ актива → запись в history
- Получить историю → показать в UI
- Убедиться, что дата и формат читаемы

- На дашборде должны появиться реальные тренды вместо "−1 за неделю"

## 📋 Волна 3: Аудит-логирование (2–3 часа)

### 6️⃣ **Аудит-лог (базовый)** — 2 часа
**Зависит от:** Assets, Tasks, Notes, Users (должны быть готовы)  
**Влияет на:** все операции (CREATE, UPDATE, DELETE)

**Что делать:**
- Backend:
  1. Новая коллекция `audit_logs` (user_id, action, entity_type, entity_id, before, after, created_at, ip_address?)
  2. Функция `log_audit(user_id, action, entity_type, entity_id, before, after)` в `backend/core/audit.py`
  3. Вызывать `log_audit()` в **каждом** POST/PUT/DELETE роуте:
     - `routers/assets.py` — CREATE, UPDATE, DELETE
     - `routers/tasks.py` — CREATE, UPDATE, DELETE
     - `routers/notes.py` — CREATE, UPDATE, DELETE
     - `routers/users.py` — CREATE, UPDATE, DELETE (особо важно)
     - `routers/categories.py` — CREATE, UPDATE, DELETE
  4. Эндпоинт `GET /api/v1/audit-logs?entity_type=assets&user_id=...&days=30` → вернуть логи с фильтрами

- Frontend (опционально для v1):
  1. Новая страница `pages/audit.js` (admin-only)
  2. Таблица логов: дата, юзер, действие, сущность, before/after
  3. Фильтры: тип сущности, юзер, диапазон дат

**Структура записи:**
```python
{
  "user_id": ObjectId,
  "action": "create" | "update" | "delete",
  "entity_type": "asset" | "task" | "note" | "user" | "category",
  "entity_id": ObjectId,
  "before": {...},  # для update: старые значения
  "after": {...},   # новые значения
  "created_at": ISODate,
  "ip_address": "192.168.1.1"  # опционально
}
```

**Тестировать:**
- Создать актив → проверить запись в audit_logs
- Изменить задачу → before/after должны быть разные
- Удалить заметку → action='delete'
- Фильтры работают
