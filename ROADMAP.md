# Roadmap

## Запланировано

- **Тёмная тема — точечная адаптация цветов категорий** — основная адаптация сделана (CSS-переменные `--border-soft`/`--border-divider`/`--row-tint`, сохранение `User.theme` через `PATCH /auth/me`). Осталось: пользовательские цвета категорий (`cat.color`) на тёмной теме могут давать низкий контраст с тёмным фоном (например жёлтый текст на чёрном — мерцает). Возможно нужно вычислять «adapted color» для тёмной темы (например через `color-mix` с `--text-main`) и применять к бейджам/точкам/тексту-в-цвет-категории.
- **«График работ»** — определить, что это (Gantt будущих задач? недельный план?), затем реализовать. Сейчас на дашборде показано реальное число запланированных, но визуализация — заглушка.
- **Вкладка «Настройки»** с подразделами по категориям (профиль, внешний вид, уведомления, интеграции, безопасность). Пункт в сайдбаре + страница `/settings` с табами слева. Логично объединить с переключателем темы.

## UX-улучшения

- **Уведомления / напоминания** — никакого видимого индикатора у срочных задач. Нужен бейдж на иконке колокольчика в шапке + список последних событий по клику. Возможно — браузерные нотификации.
- **Быстрое создание из любой вкладки** — кнопка «+» с меню «новый актив / задача / запись» в шапке или сайдбаре. Сейчас приходится идти на нужную вкладку.
- **Поиск в шапке** — сейчас просто декоративное поле. Сделать глобальный поиск по активам, сотрудникам, задачам, заметкам.
- **Информационные панели** (статистика наверху таблиц активов / сотрудников / календаря) — переработать или убрать: занимают место и дублируют дашборд.
- **Анимации фона** — тонкий движущийся градиент / параллакс / частицы. По умолчанию выкл, переключатель в настройках. Обсудить какой именно эффект.

## Dashboard — заглушки в коде

- **Метрические карточки** (`frontend/src/pages/dashboard.js` → `renderMetrics()`) — все значения и тренды захардкожены. Нужна коллекция `metrics_snapshots` (снимок каждый день) для сравнения. На фронте: `GET /api/v1/metrics?period=week` → сравнить сейчас vs неделю назад.
- **Спарклайны в карточках** (`metricCard()`) — синусоиды вместо реальных данных. Зависит от `metrics_snapshots`: 14 точек за 2 недели → нормировка в 0..50 для SVG.
- **График активности** (`renderActivityCard()`) — захардкожены. Нужен эндпоинт `GET /api/v1/analytics/activity?days=22` с агрегатом по дням (created, completed).
- **Timeline / Gantt** (`renderTimeline()`) — демо-полосы пока нет реальных задач. Исчезнут автоматически, когда есть задачи в окне 21 день.
- **Расписание (кнопка периода)** — события загружаются, но нет фильтрации по дням при клике на день в календаре.

## Бэкенд

- **Снапшоты метрик** — коллекция `metrics_snapshots` (запись каждый день: дата, кол-во активов, задач, критических). Cron-задача раз в сутки. Основа для трендов (спарклайны) и сравнения (метрические карточки).

## AI-поддержка

- **Чат-виджет онлайн-поддержки** — плавающая кнопка в углу → выезжающий чат. Бесплатный провайдер (кандидаты: Groq, Gemini free, OpenRouter free, HuggingFace). Backend `POST /api/v1/support/chat` — proxy к провайдеру (ключ только на сервере), rate-limit на пользователя. История чата в localStorage. **Правила/system prompt — обсудим позже** (контекст системы, что AI можно/нельзя).

## Идеи на подумать

- **Кастомизация аватаров** (`frontend/src/pages/users.js` → `avatarStyle()`) — сейчас цвет детерминирован по хешу username. Позволить выбирать цвет или загружать картинку. Хранить `avatar_color`/`avatar_url` в коллекции users.

## Данные и доменная логика

- **Комментарии к задаче** — отдельная коллекция или поле-массив. Полезно для переписки исполнителя с админом.
- **Файлы / вложения** — прикреплять фото техники, накладные, акты к активам и задачам. Нужен upload-эндпоинт + хранилище (S3-совместимое или локальная папка).
- **Повторяющиеся события** в календаре (еженедельная инвентаризация, ежемесячный отчёт). RRULE-подобная схема.
- **Импорт данных** (CSV → массовое создание активов / сотрудников). Парная функция к экспорту.

## Таблицы / списки

- **Массовые операции** (multi-select строк) — отметить N активов и переназначить МОЛ / удалить / экспортнуть.
- **Серверные фильтры в таблице активов** — по типу, МОЛ, статусу гарантии. Сейчас фильтры client-side по текущей странице (25 строк). На больших объёмах потребуется тянуть в бэкенд.

## Платформа / инфраструктура

- **Роли пользователей (доделать)** — модель `user/admin` решена. На бэке `require_admin` уже применён к `users`, `assets`, `tasks`, `notes`, `categories`, `calendar`. Осталось: фронт прячет admin-кнопки и закрывает страницы (см. секцию «Запланировано» → Скрытие UI).
- **Email / push-нотификации** — рассылка о приближающихся дедлайнах задач и истекающих гарантиях. Зависит от SMTP-клиента (см. «Email-инвайт»).
- **Мобильная адаптация** — сейчас сайдбар фиксированной ширины 16rem ломает раскладку на узких экранах. Бургер-меню + проверка всех страниц.
